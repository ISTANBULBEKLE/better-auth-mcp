Introduction

Model Context Protocol

The open standard for connecting AI models to data sources and tools. No more custom integrations for every data provider.

Standardized

One protocol for all AI applications.

Secure

User-controlled access to local & remote data.

Flexible

Support for Resources, Tools, and Prompts.

Architecture

The Three Pillars

MCP standardizes the communication between three key participants:

MCP Host

The IDE or App (e.g., Cursor, Claude Desktop)

↔

MCP Client

The connection manager inside the host

↔

MCP Server

The data source (SQLite, GitHub, Google Drive)

Terminology

Core Primitives

Resources

Read-only data (Databases, logs, documentation files).

Tools

Actionable functions (Run code, send email, search API).

Prompts

Reusable templates for standardized AI interaction.

Everything follows the JSON-RPC 2.0 specification.

Transports

Standard Transports

How the data actually moves between the Client and Server:

STDIO

Standard Input/Output. Used for local processes and binary servers.

SSE

Server-Sent Events. Used for remote servers over HTTP/HTTPS.
